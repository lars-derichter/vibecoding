## The Pro Path

I will walk through the full loop — not just "ask Claude for code".

<v-clicks>

1. **Create the repo on GitHub** with a README → `git clone` locally
2. **Open in VS Code**, launch Claude Code
3. **Co-create `CLAUDE.md`** — `/init`, then edit together
4. **Plan mode** for the feature (<kbd>Shift</kbd>+<kbd>Tab</kbd>)
5. **Implement** the approved plan
6. **Review the diff** — _"What else does this touch? Any regressions?"_
7. **Commit** with a good message, push back to GitHub

</v-clicks>

## What to Watch For

<v-clicks>

- How I **read the plan** before approving it — and push back
- How I keep **CLAUDE.md** in sync as the project changes
- How I **review the diff** before committing — not after
- How I write a commit message that explains _why_, not just _what_

</v-clicks>
